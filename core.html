Invariance #

Theorem doubleround_is_left_invariant is independent of the number of rounds performed.

doubleround_10_is_left_invariant is created proving the invariance for 10 rounds.

source
@[simp]
theorem core.doubleround_10_is_left_invariant (A : bitvec params.word_len) :
core.doubleround_10 (doubleround.input A) = doubleround.input A

doubleround_10 is left invariant.

Linear transformation #

Salsa20 core function can behave as a linear transformation of the form 2 * A

https://www.iacr.org/archive/fse2008/50860470/50860470.pdf

Theorem 5 of the paper.

source
@[simp]
theorem core.salsa20_core_linear_transformation (A : bitvec params.word_len) :
core.core (doubleround.input A) = 2 * doubleround.input A

def core.output (z : fin params.two_31.to_nat) :
matrixType

An hypotetical collission output of the core function where the inputs are:

Z −Z Z −Z −Z Z −Z Z Z −Z Z −Z −Z Z −Z Z

or

Z' −Z' Z' −Z' −Z' Z' −Z' Z' Z' −Z' Z' −Z' −Z' Z' −Z' Z'

Where Z and Z' are of the form of the definitions above.

Note : This type of input is not allowed in Salsa20. When expansion matrix us used, diagonal constants are added that mitigates this problem.

https://www.ecrypt.eu.org/stream/papersdir/2008/011.pdf

However, at the core function level, wthout the constants added, this inputs are possible.

Equations
  • core.output z = let x : bitvec params.word_len := core.Z z in ((2 * x, 2 * -x, 2 * x, 2 * -x), (2 * -x, 2 * x, 2 * -x, 2 * x), (2 * x, 2 * -x, 2 * x, 2 * -x), 2 * -x, 2 * x, 2 * -x, 2 * x)
source
@[simp]
theorem core.collision (z : fin params.two_31.to_nat) (h1 : core.Z z < params.two_31) (h2 : core.Z' z = core.Z z MOD params.two_31) (h3 : -core.Z z < params.two_31) (h4 : -core.Z' z = (-core.Z z) MOD params.two_31) :
core.core (doubleround.input (core.Z' z)) = core.output z ∧ core.core (doubleround.input (core.Z z)) = core.output z

Two different specially crafted inputs produces the same output.
